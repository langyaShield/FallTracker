# FallTracker 项目重构交接指南与剩余任务清单

## 1. 当前项目上下文与已完成工作

目前项目正在进行深度模块化重构。已建立清晰的领域驱动设计（DDD）分层架构，前后端均已完成主要模块的拆分。

### 1.1 已建立的后端架构规范（参考 `backend/app/modules/applications/`）

* `router.py` (如 `deliveries.py`, `events.py`)：仅作为 HTTP 适配层，禁止直接操作数据库或拼接 SQL。
* `service.py`：负责所有写操作（Create, Update, Delete）与事务管理。
* `queries.py`：负责所有只读操作（List, Get, 统计等）。
* `repository.py`：封装所有 ORM 持久化操作与级联清理。
* `csv_service.py`：独立处理导入/导出等特定领域服务。

### 1.2 已建立的前端架构规范（参考 `frontend/src/modules/applications/`）

* `types.ts`：定义领域强类型接口（Delivery、InterviewEvent、DeliveryNote 等）。
* `queries.ts`：封装所有只读 API 调用（列表、详情、统计、标签等）。
* `commands.ts`：封装所有写操作 API 调用（新增、更新、删除、批量操作等）。
* `index.ts`：统一出口，UI 组件通过 `@/modules/applications` 导入。
* UI 组件（Pages/Components）只处理交互与状态展示，禁止直接使用 `fetch` 或 `axios` 发起网络请求。

## 2. AI 执行约束与开发原则（非常重要）

在执行后续任务时，请严格遵守以下原则：

1. **环境限制**：本地环境缺少部署环境的依赖，**请仅进行代码的重构修改与本地静态语法/差异检查**。不要尝试运行 `pytest`、安装依赖或启动服务，所有运行测试将由人类在部署服务器上完成。
2. **职责分离**：
   * **后端**：Router 绝对不能直接调用数据库；复杂的业务逻辑必须下沉到 Service 层。
   * **前端**：UI 组件（Pages/Components）只处理交互与状态展示，绝对不能直接使用 `fetch` 或 `axios` 发起网络请求。
3. **渐进式提交**：请按照下列任务模块逐一推进，完成一个切片并确认无误后再进行下一个，保持代码结构的稳定性。
